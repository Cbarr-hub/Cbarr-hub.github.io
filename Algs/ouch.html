O.U.C.H.

Oversight, Underperformance, Constraints, & Hangups — An intentionally bloated corporate simulation of a heap sort algorithm.

OVERVIEW

Algorithm Summary

The O.U.C.H. protocol is a satirical simulation of an inefficient corporate environment, built on top of a classic Heapsort framework. Instead of sorting elements cleanly, this algorithm wraps every data point into a heavy middle-management object that must constantly look up its hierarchy structure. To make matters worse, it introduces randomized "Corporate Vetoes" that can completely disrupt organized sections of the pool, forcing unnecessary work and tanking algorithmic performance.

Operationally, the array acts as an organization chart. Elements are compared against their direct managers. If a subordinate out-performs their manager, a structural swap takes place to promote them. However, because of the management data overhead and random corporate interventions, the pipeline experiences intentional delays, re-orgs, and regressions before reaching finality.
